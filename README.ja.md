# プロジェクト名

このプロジェクトは、責任の範囲を明確にするためのツールです。

## インストール

```bash
# リポジトリをクローンします
git clone <repository-url>

# 依存関係をインストールします
cd responsibility-closure
npm install
```

## 使用方法

1. `examples/sample_manifest.yaml`を参考にして、マニフェストファイルを作成します。
2. コマンドラインから以下のコマンドを実行します。
   
   ```bash
   node index.js <path-to-your-manifest>
   ```

# 責任の範囲ループ (RCL)

このリポジトリは、システム設計において**判断と責任が省略されない**ことを保証するための**機械可読のフレームワーク**を提供します。

自動的に決定を行うものではありません。  
決定が明示的に人間によって所有され、完了されることを強制します。

---

## これは何か

このリポジトリには以下が含まれています：

- **最小限の責任マニフェストテンプレート** (YAML)
- **責任のギャップをレビューする固定AI監査人のプロンプト**
- **責任の完了を強制する設計フロー** (人間 → AI → 人間)

これは、実際の設計作業においてVCDesignの原則を運用化するための**ツールキット**です。

---

## これは何ではないか

- ❌ システムジェネレーター
- ❌ 最適化フレームワーク
- ❌ AI意思決定エンジン
- ❌ ベストプラクティスカタログ

このリポジトリは、*何を決定すべきか*を教えません。  
決定を**省略する行為を防ぐ**だけです。

---

## 憲法

VCDesign は、常時高速最適化という環境において、
人とシステムが生む価値を継続するために、
判断と責任の置き場を設計する思想である。
その実装態度が、再生成前提設計である。

---

## 背景哲学

私たちは、環境が次のようであると仮定します：

> **常時稼働の高速最適化**

そのような環境では：
- システムは常に再生成される
- 実装は設計意図よりも早く変化する
- AIは人間の判断を簡単にバイパスできる

VCDesignは、**判断と責任がどこに存在するかを設計する**ことでこれに対処します。

このリポジトリは、その哲学を運用形式として実装します。

---

## コアコンセプト：責任の完了ループ

すべての設計決定は、次のループを通じて表現され、レビューされます：
P → D → Δ → R

- **P (前提条件 / 約束)**  
  何が真でなければならず、自動的に決定されてはならないこと

- **D (実行 / 決定 / 表示)**  
  実行、UI、バッチ、またはAI支援（最終的な判断ではない）

- **Δ (デルタ)**  
  Pが満たされていないことを示す事実  
  （実行されていない、遅延、未確認、境界違反、証拠なし）

- **R (解決)**  
  責任の明示的な完了  
  （停止、人間に戻す、レビューをトリガー、根本原因で閉じる）

> 未解決のΔが存在する限り、設計は不完全です。

---

## 固定原則（憲法）

以下の原則は**交渉不可**です：

- デルタ（Δ）は常に解決（R）を必要とする
- 境界違反は実行を停止しなければならない
- 証拠がない場合、最終決定はできない
- 解決は省略できない
- 根本原因の完了は複数の症状のデルタを解決できる

これらの原則は**設定オプションではありません**。

---

## 設計フロー（人間 → AI → 人間）

### ステップ1：人間が責任を定義する

デザイナーは、次のことを説明する**責任マニフェスト**を書きます：
- 何の価値を保持しなければならないか
- どこで判断が許可されているか、または禁止されているか
- 進行する前に何が真でなければならないか

このステップはAIに委任できません。

---

### ステップ2：AIが監査し、決定しない

AI監査人はマニフェストをレビューし：
- 曖昧な前提条件（P）を検出する
- 不足または弱いデルタ（Δ）を検出する
- 解決（R）が省略される可能性のあるパスを検出する
- 固定原則の違反を検出する

AIは：
- ❌ 決定しない
- ❌ 解決策を提案しない
- ❌ 最適化しない

ただ**質問**を返すだけです。

---

### ステップ3：人間が責任を完了する

デザイナーは：
- 質問に答える
- リスクを受け入れるか拒否する
- 明示的に解決（R）を選択する

その後のみ、設計を進めることができます。
